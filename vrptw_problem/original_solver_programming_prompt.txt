## PROBLEM: QuickBite Fleet Scheduling

### CITY GEOGRAPHY

Five delivery zones plus one depot. Zone indices:
0 = Depot (Central)
1 = Zone A (Riverside)
2 = Zone B (Harbor)
3 = Zone C (Uptown)
4 = Zone D (Westgate)
5 = Zone E (Northgate)

Base travel time matrix (minutes, symmetric):

         Depot   A     B     C     D     E
Depot  [  0,    12,   18,   25,   30,   22  ]
A      [ 12,     0,    8,   15,   20,   14  ]
B      [ 18,     8,    0,   10,   18,   12  ]
C      [ 25,    15,   10,    0,    9,   11  ]
D      [ 30,    20,   18,    9,    0,    7  ]
E      [ 22,    14,   12,   11,    7,    0  ]

### TRAFFIC SIMULATION

Traffic multipliers by time period (applied to base matrix):

Period 1: 07:00-09:30  multiplier = 1.4  (morning peak)
Period 2: 09:30-11:30  multiplier = 1.0  (normal)
Period 3: 11:30-13:00  multiplier = 1.3  (lunch surge)
Period 4: 13:00-16:00  multiplier = 1.0  (normal)
Period 5: 16:00-18:00  multiplier = 1.5  (evening peak)

Additionally, Zone D (Westgate) has a roadworks penalty: 
add 5 minutes to any trip involving Zone D between 
08:00-12:00 regardless of traffic period.

Stochastic noise: multiply final travel time by a random 
factor drawn from Uniform(0.9, 1.1). Use a seeded 
numpy RandomState so runs are reproducible. Accept 
random_seed as a parameter (default=42).

### VEHICLES

5 vehicles, all starting at Depot (Zone 0) unless noted.

ID | Name   | Capacity | Start Zone | Shift Start | Max Hours
V0 | Alice  | 15 units | Depot      | 08:00       | 8h
V1 | Bob    | 12 units | Depot      | 08:00       | 8h
V2 | Carol  | 10 units | Depot      | 09:00       | 8h
V3 | Dave   | 15 units | Depot      | 08:00       | 8h
V4 | Eve    | 10 units | Zone E     | 09:30       | 8h

Shift start means the vehicle cannot depart before that time.
Eve starts in Zone E (index 5), not the depot.

Driver preferences (soft constraints, penalized not forbidden):
- Alice prefers not to be assigned Zone D (Westgate) — 
  add 8 min penalty per Zone D stop assigned to Alice
- Dave has been logging excess hours recently — 
  add a 15 min penalty if Dave's total shift exceeds 6.5h 
  (separate from the hard 8h limit)
- Carol starts late and prefers orders with late time windows 
  — add 5 min penalty per express order assigned to Carol

### ORDERS

Generate 30 orders deterministically using numpy seed 0.
Each order has:
- order_id: O00 to O29
- zone: randomly assigned from zones 1-5
- size: randomly drawn from [1, 2, 3, 4, 5] units
- priority: "express" with probability 0.25, else "standard"
- time_window_open: random time between 08:00 and 14:00 
  (in 30-min increments)
- time_window_close: open + random duration from 
  [60, 90, 120, 150] minutes
- service_time: 10 minutes for standard, 15 for express

Generate and print the full order table at startup so it 
is visible and reproducible.

### OBJECTIVE FUNCTION

The objective function receives a solution (vehicle route 
assignments) and returns a scalar cost to minimize.

Cost = 
    w1 * total_travel_time
  + w2 * shift_overtime_minutes
  + w3 * tw_violation_penalty
  + w4 * capacity_violation_penalty
  + w5 * workload_imbalance
  + w6 * driver_preference_penalty
  + w7 * express_lateness_penalty

Default weights (configurable via a weights dict):
  w1 = 1.0    # total travel time (minutes)
  w2 = 5.0    # per minute of shift length beyond 8h cap (summed over vehicles)
  w3 = 50.0   # per minute of time window violation (soft)
  w4 = 1000.0 # per unit of capacity overflow (hard-ish)
  w5 = 10.0   # variance of driver shift durations
  w6 = 1.0    # driver preference penalties (minutes)
  w7 = 100.0  # per express order delivered after window close

Hard constraints enforced via large penalty:
  - Capacity overflow: penalize heavily (w4)
  - Shift > 8h hard limit: add 5000 per violating vehicle

Soft constraints penalized:
  - Time window violations: penalize per minute late (w3)
  - Express orders delivered outside window (w7)
  - Driver preferences (w6)
  - Dave overtime beyond 6.5h (w5/preference overlap, 
    use w6 weight)

Route evaluation logic:
  For each vehicle:
    1. Start at vehicle's start zone at shift start time
    2. Travel to each order's zone in sequence
    3. Arrive, wait if before window_open, serve
    4. Record arrival time, check window violation
    5. Accumulate travel time, service time, load
    6. Return to depot at end (mandatory)
  
  Use get_travel_time() with the vehicle's current time 
  at each leg.

### LOCKED ASSIGNMENTS

Support a locked_assignments dict:
  { order_index: vehicle_index }

During decode, locked orders are forcibly assigned to 
their vehicle regardless of the position vector. 
Implement this cleanly so the solver still operates on 
the full vector but locked orders are overridden post-decode.
